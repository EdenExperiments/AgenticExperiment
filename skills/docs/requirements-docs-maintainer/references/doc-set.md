# Documentation Set Reference

## Canonical Documents

- `AGENTS.md`: repository directory and context map
- `Documentation/README.md`: canonical documentation index
- `Documentation/product-requirements.md`: product requirements
- `Documentation/planning-handoff.md`: planning and sequencing view
- `Documentation/feature-tracker.md`: feature readiness and dependency view
- `Documentation/decision-log.md`: decisions and open questions
- `docs/CURSOR-AGENT-HANDBOOK.md`: Cursor-first workflow guidance

## Update Pattern

- Requirement changes usually imply a tracker update.
- Planning changes usually imply a tracker update or a decision update.
- New ambiguity should be logged, not silently resolved.
- Rough notes should be normalized into canonical docs, not expanded indefinitely.

## Preferred Outputs

- Clear product language in `product-requirements.md`
- Practical sequencing context in `planning-handoff.md`
- Operational status rows in `feature-tracker.md`
- Explicit open questions and decisions in `decision-log.md`
